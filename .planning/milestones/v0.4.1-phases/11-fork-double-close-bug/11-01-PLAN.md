<objective>
Fix the double-close bug in Fork() by adding an ownership flag to collection structs and wrapping shared EFs in a close-once adapter.

Purpose: When client.Close() iterates cached collections, forked collections that share EF pointers with their parent currently close the same EF resource twice, causing panics or use-after-close errors for EFs with native resources (e.g., ONNX Runtime). This plan adds the primary ownership guard and defence-in-depth close-once wrapper.

Output: Modified collection structs with ownsEF field, new ef_close_once.go file with close-once wrappers, updated Fork() and Close() methods in both HTTP and embedded paths.
</objective>

**1. closeOnceEF struct** — wraps `embeddings.EmbeddingFunction`:
- Fields: `ef embeddings.EmbeddingFunction`, `once sync.Once`, `closed atomic.Bool`, `closeErr error`
- `Close() error`: uses `w.once.Do(func() { w.closed.Store(true); if closer, ok := w.ef.(io.Closer); ok { w.closeErr = closer.Close() } })`, returns `w.closeErr`
- `EmbedDocuments(ctx, texts)`: if `w.closed.Load()` return `nil, errors.New("embedding function is closed")`, else delegate to `w.ef.EmbedDocuments`
- `EmbedQuery(ctx, text)`: if `w.closed.Load()` return same error, else delegate to `w.ef.EmbedQuery`
- `Name() string`: delegate to `w.ef.Name()` (no closed check — metadata is safe)
- `GetConfig() EmbeddingFunctionConfig`: delegate to `w.ef.GetConfig()`
- `DefaultSpace() DistanceMetric`: delegate to `w.ef.DefaultSpace()`
- `SupportedSpaces() []DistanceMetric`: delegate to `w.ef.SupportedSpaces()`
- `UnwrapEmbeddingFunction() EmbeddingFunction`: if inner implements `EmbeddingFunctionUnwrapper`, delegate; else return `w.ef`

Add compile-time checks:
```go

    _ embeddings.EmbeddingFunction          = (*closeOnceEF)(nil)
```
