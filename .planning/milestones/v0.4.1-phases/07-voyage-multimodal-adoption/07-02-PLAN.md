**Test groups with specific test functions:**

**Group 1: Capability Derivation (VOY-01)**

`TestVoyageCapabilitiesForModel`:
- Subtest `"voyage-multimodal-3.5 returns text/image/video"`: Assert `caps.Modalities` has length 3, contains `ModalityText`, `ModalityImage`, `ModalityVideo`. Assert `caps.Intents` has length 2, contains `IntentRetrievalQuery`, `IntentRetrievalDocument`. Assert `caps.SupportsBatch == true`, `caps.SupportsMixedPart == true`. Assert `caps.SupportsRequestOption(embeddings.RequestOptionDimension) == true`.
- Subtest `"voyage-multimodal-3 returns text/image only"`: Assert `caps.Modalities` has length 2. No video. No dimension support.
- Subtest `"unknown model returns text-only"`: Assert `caps.Modalities` has length 1, equals `ModalityText`. `SupportsMixedPart == false`.

`TestVoyageCapabilities`:
- Subtest `"multimodal model returns 3 modalities"`: Create EF with model `voyage-multimodal-3.5`, call `Capabilities()`, assert 3 modalities and SupportsMixedPart true.
- Subtest `"text-only model returns 1 modality"`: Create EF with model `voyage-2`, assert 1 modality, SupportsMixedPart false.

**Group 2: Intent Mapping (VOY-02)**

`TestVoyageMapIntent`:
- Subtest `"retrieval_query maps to query"`: `MapIntent(IntentRetrievalQuery)` returns `"query"`, no error.
- Subtest `"retrieval_document maps to document"`: `MapIntent(IntentRetrievalDocument)` returns `"document"`, no error.

`TestVoyageMapIntentRejects`:
- Subtest `"classification rejected"`: `MapIntent(IntentClassification)` returns error containing `"not supported"`.
- Subtest `"clustering rejected"`: `MapIntent(IntentClustering)` returns error containing `"not supported"`.
- Subtest `"semantic_similarity rejected"`: `MapIntent(IntentSemanticSimilarity)` returns error containing `"not supported"`.
- Subtest `"custom intent rejected"`: `MapIntent(Intent("custom_thing"))` returns error containing `"unsupported intent"`.

**Group 3: Intent Resolution (VOY-02)**

`TestVoyageResolveInputType`:
- Subtest `"ProviderHints override takes priority"`: Create content with `ProviderHints: map[string]any{"input_type": "query"}` and Intent `IntentRetrievalDocument`. Assert resolved input type is `"query"` (ProviderHints wins over intent).
- Subtest `"intent mapped when no hints"`: Content with `Intent: IntentRetrievalQuery`, no ProviderHints. Assert resolved is `"query"`.
- Subtest `"default returned when no intent or hints"`: Content with no Intent, no ProviderHints. Pass `defaultInputType = &InputTypeDocument`. Assert resolved is `"document"`.
- Subtest `"nil returned when no intent, no hints, no default"`: Assert resolved is nil.

**Group 4: Content Conversion (VOY-01)**

`TestVoyageConvertToVoyageInput`:
- Subtest `"text part"`: Content with single text part "hello". Assert output has 1 block with Type="text", Text="hello".
- Subtest `"image URL part"`: Content with image part, SourceKindURL. Assert Type="image_url", ImageURL set.
- Subtest `"image base64 part"`: Content with image part, SourceKindBytes, MIMEType="image/png". Assert Type="image_base64", ImageBase64 starts with `"data:image/png;base64,"`.
- Subtest `"video URL part"`: Content with video part, SourceKindURL. Assert Type="video_url", VideoURL set.
- Subtest `"mixed text+image"`: Content with text part + image URL part. Assert 2 blocks with correct types.
- Subtest `"unsupported modality audio"`: Content with audio part. Assert error.

**Group 5: MIME Resolution**

`TestVoyageResolveMIME`:
- Subtest `"explicit MIME type used"`: Source with MIMEType="image/jpeg". Assert returns "image/jpeg".
- Subtest `"extension fallback .png"`: Source with FilePath="/path/test.png", no MIMEType. Assert "image/png".
- Subtest `"no MIME and no extension fails"`: Source with no MIMEType, no FilePath. Assert error containing "MIME type is required".

**Group 6: Multimodal URL Derivation**

`TestVoyageMultimodalURL`:
- Subtest `"default URL replaced"`: `multimodalURL("https://api.voyageai.com/v1/embeddings")` returns `"https://api.voyageai.com/v1/multimodalembeddings"`.
- Subtest `"custom URL uses hardcoded"`: `multimodalURL("https://custom.proxy.com/api")` returns the hardcoded `multimodalBaseAPI` constant.

**Group 7: Batch Rejection (VOY-01, D-16)**

`TestVoyageBatchRejectsPerItemOverrides`:
- Use httptest.NewServer that returns a mock 200 response with valid embedding data.
- Create EF with the test server URL using `WithBaseURL`, `WithAPIKey("test-key")`, `WithInsecure()`.
- Subtest `"batch rejects per-item Intent"`: Call EmbedContents with 2 items where one has Intent set. Assert error containing "per-item Intent".
- Subtest `"batch rejects per-item Dimension"`: 2 items where one has Dimension set. Assert error containing "per-item Dimension".
- Subtest `"batch rejects per-item ProviderHints input_type"`: 2 items where one has ProviderHints["input_type"]. Assert error containing "per-item ProviderHints".

**Group 8: Registration and Config Round-Trip (VOY-03)**

`TestVoyageContentRegistration`:
- Assert `embeddings.HasContent("voyageai") == true`.

`TestVoyageConfigRoundTrip`:
- Set `VOYAGE_API_KEY` env var to `"test-key-for-config"` using `t.Setenv`.
- Create EF with `NewVoyageAIEmbeddingFunction(WithAPIKey("test-key-for-config"), WithDefaultModel("voyage-multimodal-3.5"))`.
- Call `GetConfig()`, assert `cfg["model_name"] == "voyage-multimodal-3.5"` and `cfg["api_key_env_var"] == "VOYAGE_API_KEY"`.
- Reconstruct with `NewVoyageAIEmbeddingFunctionFromConfig(cfg)`, call `GetConfig()` again, assert same values.

**Group 9: Compile-time Interface Assertions (VOY-01)**

`TestVoyageInterfaceAssertions`:
- Assert `var _ embeddings.ContentEmbeddingFunction = (*VoyageAIEmbeddingFunction)(nil)` compiles. Since these are already in voyage.go, this test just confirms by building. A simple test function that calls `ef.Capabilities()` and `ef.MapIntent(embeddings.IntentRetrievalQuery)` is sufficient.
