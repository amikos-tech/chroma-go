```go
const (
    defaultMultimodalModel = "voyage-multimodal-3.5"

    Type        string `json:"type"`
    Text        string `json:"text,omitempty"`
    ImageBase64 string `json:"image_base64,omitempty"`

    VideoURL    string `json:"video_url,omitempty"`
}

type MultimodalInput struct {
    Content []MultimodalContentBlock `json:"content"`
}

type CreateMultimodalEmbeddingRequest struct {
    Model           string            `json:"model"`
    Inputs          []MultimodalInput `json:"inputs"`
    InputType       *InputType        `json:"input_type"`

    OutputDimension *int              `json:"output_dimension,omitempty"`
}
```

**2. Extension-to-MIME map (copied from Gemini pattern, scoped to Voyage-supported formats per D-11, D-12):**

```go
var extToMIME = map[string]string{
```

**3. `capabilitiesForModel(model string) embeddings.CapabilityMetadata` (per D-09, D-10, D-17, D-18, D-19, D-23):**

Switch on model:
- `"voyage-multimodal-3.5"`: Modalities=[text, image, video], Intents=[retrieval_query, retrieval_document], RequestOptions=[RequestOptionDimension], SupportsBatch=true, SupportsMixedPart=true
- `"voyage-multimodal-3"`: Modalities=[text, image], Intents=[retrieval_query, retrieval_document], SupportsBatch=true, SupportsMixedPart=true (no dimension support — older model)
- default: Modalities=[text], SupportsBatch=true (text-only models)

**4. Binary resolution helpers (copied from Gemini content.go pattern per research "Don't Hand-Roll"):**

- `resolveBytes(ctx context.Context, source *embeddings.BinarySource, maxFileSize int64) ([]byte, error)` — handles SourceKindBytes, SourceKindBase64, SourceKindFile, rejects SourceKindURL
- `resolveMIME(source *embeddings.BinarySource) (string, error)` — MIMEType field first, then file extension fallback, then explicit error
- `containsDotDot(path string) bool` — path traversal check
- `validateMIMEModality(modality embeddings.Modality, mimeType string) error` — image/* for image, video/* for video

**5. Content-to-Voyage conversion (per D-05, D-06, D-07, D-08):**

- `convertToVoyageInput(ctx context.Context, content embeddings.Content, maxFileSize int64) (*MultimodalInput, error)` — iterates content.Parts, calls buildContentBlock for each
- `buildContentBlock(ctx context.Context, part embeddings.Part, maxFileSize int64) (MultimodalContentBlock, error)` — switches on part.Modality:
  - ModalityText: `{Type: "text", Text: part.Text}`
  - ModalityImage: calls `buildBinaryBlock(ctx, part.Source, "image", maxFileSize)`
  - ModalityVideo: calls `buildBinaryBlock(ctx, part.Source, "video", maxFileSize)`
  - default: returns error
- `buildBinaryBlock(ctx context.Context, source *embeddings.BinarySource, mediaType string, maxFileSize int64) (MultimodalContentBlock, error)`:
  - If SourceKindURL: type=`mediaType+"_url"`, set ImageURL or VideoURL field
  - Otherwise: resolveMIME, validateMIMEModality, resolveBytes, encode as data URI `data:<mime>;base64,<encoded>`, type=`mediaType+"_base64"`, set ImageBase64 or VideoBase64

**6. Intent mapping (per D-13, D-14, D-15):**

- `MapIntent(intent embeddings.Intent) (string, error)` on VoyageAIEmbeddingFunction:
  - If `!embeddings.IsNeutralIntent(intent)`: return error "unsupported intent %q: use ProviderHints[\"input_type\"] for Voyage-native input types"
  - `IntentRetrievalQuery` -> `string(InputTypeQuery)` (="query")
  - `IntentRetrievalDocument` -> `string(InputTypeDocument)` (="document")
  - All others (classification, clustering, semantic_similarity): return error "intent %q is not supported by Voyage; only retrieval_query and retrieval_document are available"

**7. `resolveInputTypeForContent(content embeddings.Content, defaultInputType *InputType, mapper embeddings.IntentMapper) (*InputType, error)` (per D-15):**

Priority chain:
1. `content.ProviderHints["input_type"]` -> parse as string, create `*InputType`
2. `content.Intent` != "" and mapper != nil -> call `mapper.MapIntent(content.Intent)`, create `*InputType`
3. Return defaultInputType

**8. Capabilities() on VoyageAIEmbeddingFunction (per D-09, D-23):**

```go
func (e *VoyageAIEmbeddingFunction) Capabilities() embeddings.CapabilityMetadata {
    return capabilitiesForModel(string(e.apiClient.DefaultModel))
}
```

**9. EmbedContent and EmbedContents on VoyageAIEmbeddingFunction (per D-16, D-27):**

EmbedContent:
- Validate content, validate support against capabilities, call CreateMultimodalEmbedding with single-item slice, return first result

EmbedContents:
- ValidateContents, check max batch size, ValidateContentsSupport
- For batch (len > 1): reject per-item Intent, Dimension, ProviderHints["input_type"] with explicit errors (per D-16)
- For single item: allow per-item overrides
- Resolve input type (single-item per-item override, batch uses default)
- Resolve dimension (single-item per-item override, batch uses nil unless context override)
- Convert all contents to []MultimodalInput
- Call CreateMultimodalEmbedding

**10. `multimodalURL(baseAPI string) string` helper (per Pitfall 4 and Pitfall 6):**

Derive multimodal endpoint URL from BaseAPI:
- If baseAPI ends with `/v1/embeddings`, replace with `/v1/multimodalembeddings`
- Otherwise, use hardcoded `multimodalBaseAPI` constant
