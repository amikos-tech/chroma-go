Fix Phase 24 so embedded `GetOrCreateCollection(...)` never passes a closed EF into the `CreateCollection(...)` fallback and concurrent miss/create races still return usable collections. The fix must preserve Phase 20 existing-state precedence and Phase 23’s narrow reload/default-cleanup behavior while making provisional `GetCollection(...)` cleanup ownership-aware across dense EF, `contentEF`, and dual-interface content EF paths.

Output: ownership-aware embedded state cleanup, conditional convergence for concurrent losers, deterministic dense/content/dual-interface fallback regressions, a `-race` miss/create regression, and a green V2 test/lint pass.
