From pkg/api/v2/client_http.go (GetCollection, lines ~422-463):
- Auto-wires contentEF first via BuildContentEFFromConfig(configuration)
- Then auto-wires dense EF: tries unwrapping from contentEF, then BuildEmbeddingFunctionFromConfig
- Wraps both in close-once wrappers

From pkg/api/v2/configuration.go:
- BuildEmbeddingFunctionFromConfig(config) -> builds dense EF from stored config
- BuildContentEFFromConfig(config) -> builds content EF from stored config with fallback chain
- Config keys: "embedding_function.type", "embedding_function.name"
