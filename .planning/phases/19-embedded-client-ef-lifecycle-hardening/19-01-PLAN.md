**CRITICAL DESIGN NOTE (addresses Codex review concern #3 — close-once wrapper storage location):**
The `collectionState` map must store close-once WRAPPED EFs so that `deleteCollectionState`, `embeddedLocalClient.Close()`, and `embeddedCollection.Close()` all close through the SAME `sync.Once` wrapper. Without this, `deleteCollectionState` closing a raw EF and the collection closing through its close-once wrapper would double-close the underlying resource. `wrapEFCloseOnce` and `wrapContentEFCloseOnce` are idempotent (if input is already wrapped, return unchanged), so `buildEmbeddedCollection` calling them again is safe — it gets the same wrapper back.

The write lock spans ONLY the check-nil + build-EF + assign cycle — it does NOT span the subsequent `buildEmbeddedCollection` call (which would deadlock since `buildEmbeddedCollection` calls `upsertCollectionState` internally).

**3. deleteCollectionState closes EFs then removes entry (client_local_embedded.go, ~lines 681-688) per D-06:**

(Addresses Codex review concern #1 — delete/close ordering: EFs are closed BEFORE the map entry ceases to exist from the caller's perspective. The implementation copies the state ref and deletes the entry under lock, then closes outside the lock. The delete-under-lock prevents a concurrent GetCollection from seeing stale EFs, while closing outside the lock avoids blocking. Since collectionState stores close-once wrapped EFs, this close path shares the same wrapper as any live embeddedCollection — sync.Once prevents double-close.)

Replace the current implementation:
```go
func (client *embeddedLocalClient) deleteCollectionState(collectionID string) {
    if collectionID == "" {
        return
    }
    client.collectionStateMu.Lock()
    state := client.collectionState[collectionID]
    delete(client.collectionState, collectionID)
    client.collectionStateMu.Unlock()
    if state != nil {
        if err := closeEmbeddingFunctions(state.embeddingFunction, state.contentEmbeddingFunction); err != nil {
            logCollectionCleanupCloseErrorToStderr(collectionID, err)
        }
    }
}
```

The `closeEmbeddingFunctions` call operates on close-once wrapped EFs from state. If an `embeddedCollection` also holds the same wrappers, `sync.Once` ensures the underlying EF is closed at most once.

**4. embeddedLocalClient.Close() iterates collectionState (client_local_embedded.go, ~lines 648-665) per D-05:**

(Addresses Codex review concern #2 — Client Close() ordering: EFs are closed first by iterating a snapshot of collectionState, then the map is cleared. The runtime and state client are closed afterward. Uses errors.Join for complete error aggregation per Gemini review concern #9.)

Replace the current `Close()` method with:
```go
func (client *embeddedLocalClient) Close() error {
    var errs []error

    // D-05: Close all tracked EFs before shutting down runtime.
    // Copy state under lock, clear map, release lock, then close outside.
    client.collectionStateMu.Lock()
    states := make(map[string]*embeddedCollectionState, len(client.collectionState))
    for k, v := range client.collectionState {
        states[k] = v
    }
    client.collectionState = map[string]*embeddedCollectionState{}
    client.collectionStateMu.Unlock()

    for id, s := range states {
        if err := closeEmbeddingFunctions(s.embeddingFunction, s.contentEmbeddingFunction); err != nil {
            logCollectionCleanupCloseErrorToStderr(id, err)
            errs = append(errs, err)
        }
    }

    if client.state != nil {
        if err := client.state.Close(); err != nil {
            errs = append(errs, err)
        }
    }
    if client.embedded != nil {
        if err := client.embedded.Close(); err != nil {
            errs = append(errs, errors.Wrap(err, "error closing embedded local runtime"))
        }
    }
    if len(errs) > 0 {
        return stderrors.Join(errs...)
    }
    return nil
}
```

Note: `stderrors` is already imported as an alias for the standard `errors` package in this file. The state's EFs are close-once wrapped so the `sync.Once` prevents double-close with any live `embeddedCollection` objects. The loop continues closing all entries even if one fails (addresses Gemini review concern #9 — error aggregation).
