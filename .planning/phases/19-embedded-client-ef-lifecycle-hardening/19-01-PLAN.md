**Fix 3: deleteCollectionState closes EFs then removes entry (client_local_embedded.go, ~lines 681-688) per D-06:**

Replace the current implementation with:
```go
func (client *embeddedLocalClient) deleteCollectionState(collectionID string) {
    if collectionID == "" {
        return
    }
    client.collectionStateMu.Lock()
    state := client.collectionState[collectionID]
    delete(client.collectionState, collectionID)
    client.collectionStateMu.Unlock()
    if state != nil {
        if err := closeEmbeddingFunctions(state.embeddingFunction, state.contentEmbeddingFunction); err != nil {
            logCollectionCleanupCloseErrorToStderr(collectionID, err)
        }
    }
}
```

**Why this satisfies D-06 ("close before remove"):** From the caller's perspective, after `deleteCollectionState` returns, both the map entry AND the EFs are cleaned up. The map entry is removed under lock first so concurrent `GetCollection` cannot see stale EFs, while the actual `Close()` happens outside the lock to avoid blocking. The close-once wrapper shared with any live `embeddedCollection` ensures `sync.Once` prevents double-close. A concurrent `GetCollection` arriving after the delete will create a fresh state entry with fresh EFs.

**Fix 4: embeddedLocalClient.Close() iterates collectionState (client_local_embedded.go, ~lines 648-665) per D-05:**

Replace the current `Close()` method with:
```go
func (client *embeddedLocalClient) Close() error {
    var errs []error

    // D-05: Close all tracked EFs before shutting down runtime.
    // Copy state under lock, clear map, release lock, then close outside.
    client.collectionStateMu.Lock()
    states := make(map[string]*embeddedCollectionState, len(client.collectionState))
    for k, v := range client.collectionState {
        states[k] = v
    }
    client.collectionState = map[string]*embeddedCollectionState{}
    client.collectionStateMu.Unlock()

    for id, s := range states {
        if err := closeEmbeddingFunctions(s.embeddingFunction, s.contentEmbeddingFunction); err != nil {
            logCollectionCleanupCloseErrorToStderr(id, err)
            errs = append(errs, err)
        }
    }

    if client.state != nil {
        if err := client.state.Close(); err != nil {
            errs = append(errs, err)
        }
    }
    if client.embedded != nil {
        if err := client.embedded.Close(); err != nil {
            errs = append(errs, errors.Wrap(err, "error closing embedded local runtime"))
        }
    }
    if len(errs) > 0 {
        return stderrors.Join(errs...)
    }
    return nil
}
```

Note: `stderrors` is already imported as an alias for the standard `errors` package. EFs are closed BEFORE runtime shutdown. The loop continues closing all entries even if one fails.
