**IMPORTANT DESIGN NOTE (addresses Codex review concern #4 — logger propagation gap):**
`localDeleteCollectionFromCache` runs on `APIClientV2` (the state client), not on `embeddedLocalClient`. The `APIClientV2` has its own `logger` field (from `BaseAPIClient`). To ensure the injected logger reaches errors from that path, `WithPersistentLogger` must ALSO forward the logger to the state client via `WithPersistentClientOption(WithLogger(l))`. This is NOT a duplicate API surface (addresses Codex review concern #6 — WithPersistentLogger vs WithPersistentClientOption(WithLogger(...))): `WithPersistentLogger` is a convenience that sets BOTH the embedded client's logger AND the state client's logger in one call. Users CAN use `WithPersistentClientOption(WithLogger(l))` to set only the state client's logger, but `WithPersistentLogger` is the recommended single entry point.

**2. Add WithPersistentLogger option (client_local.go):**

Add after the existing `WithPersistentClientOptions` function:

```go
// WithPersistentLogger sets a structured logger for the persistent client.
// When set, auto-wire and close errors route through the logger instead of stderr.
// The logger is propagated to both the embedded client and the internal state client.
func WithPersistentLogger(l logger.Logger) PersistentClientOption {
    return func(cfg *localClientConfig) error {
        cfg.logger = l
        // Propagate to the state client so localDeleteCollectionFromCache
        // errors are also captured by the structured logger.
        cfg.clientOptions = append(cfg.clientOptions, WithLogger(l))
        return nil
    }
}
```

This ensures the logger reaches `localDeleteCollectionFromCache` (which runs on `APIClientV2` and uses `client.logger`).
